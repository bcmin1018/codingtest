## 1. Design Linked List

> LeetCode · Linked List · Singly Linked List 직접 구현
> 풀이 파일: [../linked_list/design_linked_list.py](../linked_list/design_linked_list.py)

### 문제 정의

연결 리스트를 직접 설계해 `get / addAtHead / addAtTail / addAtIndex / deleteAtIndex`를 지원. index는 0-based, 범위 밖 요청은 무시(또는 -1).

### 핵심 아이디어: Sentinel(더미) 헤드

```python
self.head = ListNode(0)  # sentinel node
```

- 데이터 없는 **가짜 머리 노드**를 항상 하나 둠.
- "맨 앞 삽입/삭제"가 중간 케이스와 **완전히 같은 코드**로 처리됨 → 경계 분기 제거.

`get_pred(index)` 헬퍼 하나로 삽입·삭제·조회에서 "건드릴 위치의 바로 앞 노드"를 공통으로 구함.
- `get(index)`의 실제 값은 `get_pred(index + 1).val` ← **+1 오프셋 주의**.

### 삽입 시 포인터 재연결 순서

```python
new_node.next = pred.next   # 1) 새 노드가 뒤를 가리키게 (잃어버릴 참조 먼저 확보)
pred.next = new_node        # 2) 앞 노드가 새 노드를 가리키게
```

`pred.next`를 먼저 덮으면 뒤 노드를 잃음. **"새 노드 next 먼저 → 앞 노드 next 나중"**.

### 복잡도

- get / addAtIndex / deleteAtIndex: O(index)
- addAtHead: O(1) / addAtTail: O(n) (tail 포인터 없음 → 개선 여지)

### 복습 포인트

- Sentinel 노드는 연결 리스트 경계 분기 제거의 핵심 패턴.
- 포인터 재연결은 **잃어버릴 참조를 먼저 확보**한 뒤 끊기.
- index 기준이 "그 노드"인지 "그 앞 노드"인지 (+1) 헷갈리지 않기.

## 2. Diagonal Traverse

> LeetCode · Array and String · 2D 행렬 시뮬레이션
> 풀이 파일: [../array_and_string/diagonal_traverse.py](../array_and_string/diagonal_traverse.py)

### 문제 정의

`m x n` 행렬을 대각선 방향 지그재그(↗↙)로 순회한 결과를 1차원 배열로 반환.

```
[[1,2,3],[4,5,6],[7,8,9]] → [1,2,4,7,5,3,6,8,9]
```

### 핵심 관찰: 대각선 번호 `row + col`

같은 대각선 위 칸들은 **`row + col`이 모두 같음.**
- 짝수 → **위로 ↗** (row 감소, col 증가)
- 홀수 → **아래로 ↙** (row 증가, col 감소)

### 벽 처리 (가장 헷갈리는 부분)

방향 전환 시 **코너를 두 번 세지 않도록 우선순위** 주의. "이동 방향 끝 벽을 먼저 체크".

```python
# 짝수 (위로 ↗):  오른쪽 벽 먼저 → 위쪽 벽 → 아니면 ↗
if col == w-1:   row += 1
elif row == 0:   col += 1
else:            row -= 1; col += 1

# 홀수 (아래로 ↙): 아래쪽 벽 먼저 → 왼쪽 벽 → 아니면 ↙
if row == h-1:   col += 1
elif col == 0:   row += 1
else:            row += 1; col -= 1
```

### 복잡도

- 시간 O(m·n) (모든 칸 1번씩), 공간 O(1) (결과 제외)

### 복습 포인트

- **대각선 = `row + col` 상수** 불변식이 핵심 (반대 대각선은 `row - col` 상수).
- 방향 전환 시 **코너 우선순위**가 버그 주범.
- 대안: 대각선별로 묶은 뒤 홀수 대각선만 reverse하는 풀이도 있음 → 비교해볼 것.
