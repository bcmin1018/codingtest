# Replace Elements with Greatest Element on Right Side 풀이 정리

> LeetCode · Fun with Arrays · In-Place Operations

## 1. 문제 정의

정수 배열 `arr`가 주어졌을 때, 각 원소를 **자기 오른쪽에 있는 원소 중 가장 큰 값**으로 교체하는 문제입니다. 마지막 원소는 오른쪽이 없으므로 `-1`로 교체합니다.

```
입력: [17, 18, 5, 4, 6, 1]
출력: [18,  6, 6, 6, 1, -1]
```

### 엣지 케이스

| 입력 | 결과 | 이유 |
|---|---|---|
| `[400]` | `[-1]` | 원소가 하나뿐이므로 오른쪽이 없음 |
| `[1, 2]` | `[2, -1]` | 마지막 원소는 항상 -1 |

---

## 2. 접근 방법

### 핵심 관찰: 오른쪽에서 왼쪽으로 순회

각 위치 `i`의 답은 **오른쪽(i+1 ~ 끝)** 에만 의존합니다. 왼쪽은 전혀 관계가 없습니다.

왼쪽→오른쪽으로 순회하면, `i=0`을 처리할 때 오른쪽 전체를 매번 다시 훑어야 해서 **O(n²)** 이 됩니다.

오른쪽→왼쪽으로 순회하면, `max_right` 변수 하나로 **"지금까지 본 오른쪽 최댓값"** 을 누적해서 들고 다닐 수 있습니다. 위치 `i`에 도달했을 때 이미 `i+1 ~ 끝`을 모두 본 상태이므로, 추가 스캔 없이 `max_right` 하나만으로 답을 구할 수 있습니다.

### 순회 흐름

```
[17, 18, 5, 4, 6, 1]

i=5: max_right = -1  →  arr[5] = -1,  max_right = max(-1, 1) = 1
i=4: max_right = 1   →  arr[4] = 1,   max_right = max(1, 6) = 6
i=3: max_right = 6   →  arr[3] = 6,   max_right = max(6, 4) = 6
i=2: max_right = 6   →  arr[2] = 6,   max_right = max(6, 5) = 6
i=1: max_right = 6   →  arr[1] = 6,   max_right = max(6, 18) = 18
i=0: max_right = 18  →  arr[0] = 18,  max_right = max(18, 17) = 18

결과: [18, 6, 6, 6, 1, -1] ✅
```

### 구현

```python
class Solution:
    def replaceElements(self, arr: List[int]) -> List[int]:
        max_ = -1
        for i in range(len(arr)-1, -1, -1):
            if arr[i] > max_:
                n_max = arr[i]
            arr[i] = max_
            max_ = n_max
        return arr
                        
```

**핵심 포인트:**
- `arr[i]`를 덮어쓰기 전에 원본값을 `new_max` 계산에 먼저 사용합니다.
- `max_right`를 갱신할 때는 덮어쓰기 전 원본 `arr[i]`와 비교해야 하므로 순서가 중요합니다.

---

## 3. 시간 복잡도 · 공간 복잡도

- **시간 복잡도: O(n)** — 배열을 정확히 한 번 역순으로 순회합니다.
- **공간 복잡도: O(1)** — `max_right`, `new_max` 변수 두 개만 사용하며, in-place로 원본 배열을 수정합니다.

### 왜 O(n)이 최적인가?

모든 원소를 최소 한 번은 확인해야 하므로 O(n)이 이론적 하한입니다. 이 풀이는 정확히 n번만 순회하므로 점근적으로 최적입니다.

| 접근 방법 | 시간 복잡도 | 공간 복잡도 |
|---|---|---|
| 왼→오 이중 루프 (Naive) | O(n²) | O(1) |
| **오→왼 단일 루프 (최적)** | **O(n)** | **O(1)** |

## 정리: 오른쪽→왼쪽 패턴을 쓰는 이유

> **"현재 위치의 답이 오른쪽(미래)에 의존할 때"** 오른쪽에서 출발합니다.

이 문제가 그 전형적인 예시입니다. 오른쪽에서 출발해 누적 최댓값을 들고 오면, 왼쪽을 다시 볼 이유가 전혀 없습니다.
